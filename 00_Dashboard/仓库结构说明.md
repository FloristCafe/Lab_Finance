# Lab_Finance 仓库结构说明

## 总原则

- 这里只放学习资料、章节笔记、技术笔记、联立思考和研究问题。
- 不放量化系统代码。
- 一章先读透，再写笔记，再做跨书联立。

## 主线目录

- `01_Harris_Market_Microstructure`：市场微观结构、订单、撮合、流动性、交易制度。
- `02_Ilmanen_Expected_Returns`：预期收益、风险溢价、资产定价、长期回报来源。
- `03_Narang_Inside_the_Black_Box`：量化研究框架、策略类型、系统视角、执行与风险。

## 书内结构

每本书都按同一套子目录写：

- `00_Reading_Plan`：阅读顺序、进度、待读章节。
- `01_Chapter_Notes`：每章一篇主笔记。
- `02_Concepts`：概念卡、术语卡。
- `03_Mechanisms`：机制拆解、流程图、状态转移。
- `04_Questions`：未解决问题、疑点、反例。
- `05_Quant_Links`：这一章和后续量化研究的连接点。

## 联立层

- `04_Cross_Book_Synthesis`：三本书之间的共同概念、差异、对照表。
- `05_Technical_Notes`：偏实现思维的学习笔记，例如 `Order`、`OrderBook`、`State Machine`、`Slippage`、`Margin`、`Telemetry`。
- `06_Glossary`：统一术语表，中英对照。
- `07_Reading_Notes`：补充文章、研报、课程、论文笔记。
- `08_Research_Ideas`：未来实践提示、研究假设、待验证问题。

## 书与内容怎么放

- 书的章节理解，放 `01_Chapter_Notes`。
- 重要定义，放 `02_Concepts`。
- 机制图和过程图，放 `03_Mechanisms`。
- 想不通的问题，放 `04_Questions`。
- 章节内容以后怎么服务量化研究，放 `05_Quant_Links`。
- 跨书共同结论，放 `04_Cross_Book_Synthesis`。
- 像“订单类字段”“撮合状态机”“滑点计算”这种偏技术的学习笔记，放 `05_Technical_Notes`，但不放代码。

## 建议的阅读顺序

- `Harris`：第 4 章 -> 第 5 章 -> 第 3 章 -> 第 13 章 -> 第 10 章 -> 第 19 章
- `Ilmanen`：后续按“收益来源 -> 风险溢价 -> 资产定价 -> 因子解释”推进
- `Narang`：按“策略框架 -> 系统模块 -> 研究/执行/风控”推进

## 笔记节奏

- 先读。
- 再写一篇章节总结。
- 再写一篇机制笔记。
- 再写一篇问题笔记。
- 最后补一篇 `Quant Link` 或 `Cross-Book` 笔记。

## 你后面最常用的联立方向

- `订单属性` -> `订单簿` -> `撮合` -> `滑点`
- `市场结构` -> `流动性` -> `执行成本`
- `风险溢价` -> `预期收益` -> `资产定价`
- `量化框架` -> `数据` -> `信号` -> `执行` -> `风控`
